# Windows 文件阻止问题 - 完整解决方案

## ❓ 问题说明

当你尝试运行下载的 `.bat` 文件时，Windows 可能会显示：
- "此文件已被阻止"
- "Windows 已保护你的电脑"
- "无法验证发布者"

这是 Windows 的安全保护机制，是正常现象。

---

## ✅ 解决方案（按推荐顺序）

---

### 方案一：使用解除阻止工具（最简单）

1. **找到文件**
   在解压后的文件夹中，找到：
   ```
   解除阻止.bat
   ```

2. **右键运行**
   - 右键点击 `解除阻止.bat`
   - 选择"以管理员身份运行"

3. **完成**
   工具会自动解除所有文件的阻止

---

### 方案二：手动解除文件锁定

1. **右键点击被阻止的文件**
   例如：`一键部署.bat`

2. **选择"属性"**

3. **解除锁定**
   - 在属性窗口底部，找到"安全"部分
   - 勾选"解除锁定"（Unblock）
   - 点击"确定"

4. **现在可以正常运行了！**

---

### 方案三：复制文件（最简单有效）

1. **复制文件**
   - 选中被阻止的文件（如 `一键部署.bat`）
   - 按 `Ctrl+C` 复制

2. **粘贴文件**
   - 按 `Ctrl+V` 粘贴
   - 会生成 `一键部署 - 副本.bat`

3. **运行副本**
   - 双击运行副本文件
   - 副本文件通常不会被阻止

---

### 方案四：使用 PowerShell 版本（推荐）

我们为你准备了 PowerShell 版本，通常不会被阻止：

1. **找到文件**
   ```
   一键部署.ps1
   ```

2. **右键运行**
   - 右键点击
   - 选择"使用 PowerShell 运行"

3. **如果提示执行策略**
   按 Y 确认执行

---

### 方案五：临时关闭 SmartScreen（不推荐，仅测试用）

1. **打开 Windows 安全中心**
   - 按 `Win + S` 搜索"Windows 安全中心"

2. **应用和浏览器控制**
   - 点击左侧"应用和浏览器控制"

3. **Reputation-based protection 设置**
   - 点击"Reputation-based protection 设置"

4. **临时关闭**
   - 关闭"检查应用和文件"
   - 运行完脚本后记得重新开启！

## 📋 如果所有方案都不行

### 手动部署步骤（最可靠）

#### 1. 安装 Node.js
- 访问：https://nodejs.org/
- 下载 LTS 版本
- 双击安装，一路下一步

#### 2. 打开命令行
- 按 `Win + R`
- 输入：`cmd`
- 按回车

#### 3. 进入项目目录
```cmd
cd C:\path\to\your\project
```

#### 4. 安装 pnpm
```cmd
npm install -g pnpm
```

#### 5. 安装依赖
```cmd
pnpm install
```

#### 6. 构建项目
```cmd
pnpm build
```

#### 7. 启动服务
```cmd
pnpm start
```

#### 8. 访问系统
打开浏览器：http://localhost:3000
