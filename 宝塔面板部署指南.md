## 前置准备

### 1. 阿里云服务器要求

- **操作系统**：Ubuntu 20.04/22.04 或 CentOS 7/8
- **配置建议**：2核4G 以上
- **宝塔面板**：已安装并正常运行
- **域名**（可选）：用于 HTTPS 访问

### 2. 本地环境

- Windows 10/11 或 macOS/Linux
- PowerShell（Windows）或终端（Mac/Linux）

### 3. 数据库准备（可选）

系统支持两种模式：
- **Mock 模式**：无需数据库，快速体验
- **Supabase 模式**：需要 Supabase 账号（免费）

如需使用 Supabase：
1. 访问 https://supabase.com 注册账号
2. 创建新项目
3. 获取 Project URL 和 Anon Key

## 方法一：一键自动部署（推荐）

### Windows 用户

#### 1. 打包项目

在项目根目录打开 PowerShell，运行：

```powershell
.\package-for-baota.ps1
```

脚本会自动：
- 打包项目文件（排除 node_modules 等）
- 询问是否上传到服务器
- 提供详细的后续操作指引

#### 2. 在服务器上部署

SSH 登录服务器后执行：

```bash
# 进入项目目录
cd /www/wwwroot/warehouse-system

# 赋予执行权限
chmod +x baota-deploy.sh

# 运行部署脚本
sudo ./baota-deploy.sh
```

脚本会自动：
- ✅ 检查并安装 Docker 环境
- ✅ 配置环境变量
- ✅ 创建 Docker Compose 配置
- ✅ 构建并启动服务
- ✅ 提供宝塔面板配置指引

### Linux/Mac 用户

#### 1. 打包并上传

```bash
# 赋予执行权限
chmod +x package-for-aliyun.sh

# 运行打包脚本
./package-for-aliyun.sh
```

#### 2. 在服务器上部署

同上，运行 `baota-deploy.sh`

## 方法二：手动部署

### 步骤 1：上传项目文件

**方法 A：使用宝塔面板文件管理器**

1. 登录宝塔面板
2. 点击左侧菜单 "文件"
3. 进入 `/www/wwwroot/` 目录
4. 点击 "上传" 按钮
5. 将整个项目文件夹上传
6. 右键上传的文件夹，选择 "解压"

**方法 B：使用 FTP/SFTP**

使用 FileZilla 或 WinSCP 连接服务器：
- 主机：服务器 IP
- 端口：22（SSH）
- 用户名：root
- 密码：你的服务器密码

上传到 `/www/wwwroot/warehouse-system`

**方法 C：使用命令行**

```bash
# 在本地项目目录
scp -r ./项目目录 root@服务器IP:/www/wwwroot/warehouse-system
```

### 步骤 2：安装 Docker

如果宝塔面板未安装 Docker：

```bash
# 方法 1：通过宝塔面板安装
# 宝塔面板 → 软件商店 → 搜索 "Docker" → 安装

# 方法 2：命令行安装
curl -fsSL https://get.docker.com | bash -s docker
systemctl start docker
systemctl enable docker
```

### 步骤 3：配置环境变量

```bash
cd /www/wwwroot/warehouse-system

# 创建 .env 文件
cat > .env << EOF
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
PORT=3000
NODE_ENV=production
TZ=Asia/Shanghai
EOF
```

### 步骤 4：启动服务

```bash
# 构建并启动
docker compose up -d --build

# 查看日志
docker compose logs -f

# 查看状态
docker compose ps
```

## 宝塔面板配置

### 1. 添加网站

1. 登录宝塔面板
2. 点击 "网站" → "添加站点"
3. 填写信息：
   - **域名**：你的域名或服务器 IP
   - **根目录**：`/www/wwwroot/warehouse-system`
   - **PHP 版本**：纯静态
   - **数据库**：不创建
4. 点击 "提交"

### 2. 配置反向代理

1. 进入网站设置（点击刚创建的网站）
2. 点击 "反向代理" 标签
3. 点击 "添加反向代理"
4. 填写：
   - **代理名称**：`warehouse-system`
   - **目标 URL**：`http://127.0.0.1:3000`
   - **发送域名**：`$host`
5. 点击 "提交"
6. 确保反向代理状态为 "开启"

### 3. 开放端口

1. 点击左侧菜单 "安全"
2. 添加端口规则：
   - **端口**：3000
   - **协议**：TCP
   - **备注**：仓库管理系统
3. 如果使用域名访问，确保 80 和 443 端口已开放

### 4. 测试访问

访问 `http://你的域名或IP`

默认登录信息：
- 用户名：`admin`
- 密码：`admin123`

⚠️ **首次登录后请立即修改密码！**

## SSL 证书配置

### 使用宝塔面板申请免费证书

1. 进入网站设置
2. 点击 "SSL" 标签
3. 选择 "Let's Encrypt"
4. 勾选你的域名
5. 点击 "申请"
6. 申请成功后，开启 "强制 HTTPS"

### 使用命令行（可选）

```bash
# 安装 certbot
apt install certbot python3-certbot-nginx -y

# 申请证书
certbot --nginx -d your-domain.com

# 自动续期已自动配置
```

## 日常运维管理

### 使用管理脚本

```bash
cd /www/wwwroot/warehouse-system
chmod +x docker-manage.sh
./docker-manage.sh
```

管理脚本提供以下功能：
1. 查看服务状态
2. 查看实时日志
3. 重启服务
4. 停止服务
5. 启动服务
6. 更新服务（重新构建）
7. 查看容器资源使用
8. 清理无用镜像
9. 备份数据
10. 进入容器终端

### 常用 Docker 命令

```bash
# 查看服务状态
cd /www/wwwroot/warehouse-system
docker compose ps

# 查看日志
docker compose logs -f

# 重启服务
docker compose restart

# 停止服务
docker compose down

# 更新服务
docker compose up -d --build

# 查看资源使用
docker stats warehouse-system

# 进入容器
docker exec -it warehouse-system /bin/sh
```

### 数据备份

```bash
# 手动备份
cd /www/wwwroot/warehouse-system
tar -czf /www/backup/warehouse-backup-$(date +%Y%m%d).tar.gz .env docker-compose.yml data/

# 使用管理脚本备份
./docker-manage.sh
# 选择选项 9
```

### 更新系统

```bash
cd /www/wwwroot/warehouse-system

# 1. 上传最新代码（覆盖现有文件）

# 2. 重新构建并启动
docker compose down
docker compose up -d --build

# 3. 查看日志确认正常
docker compose logs -f
```

## 常见问题

### 1. Docker 构建失败

**问题**：`docker compose up -d --build` 报错

**解决方案**：
```bash
# 清理 Docker 缓存
docker system prune -a

# 重新构建
docker compose build --no-cache
docker compose up -d
```

### 2. 端口被占用

**问题**：端口 3000 已被占用

**解决方案**：
```bash
# 查看端口占用
lsof -i:3000
# 或
netstat -tlnp | grep 3000

# 方法 1：停止占用端口的服务
kill -9 <PID>

# 方法 2：修改应用端口
# 编辑 .env 文件，修改 PORT=3001
# 编辑 docker-compose.yml，修改端口映射为 "3001:3000"
# 重启服务
docker compose down && docker compose up -d
```

### 3. 无法访问网站

**排查步骤**：

```bash
# 1. 检查容器是否运行
docker compose ps

# 2. 检查容器日志
docker compose logs warehouse-system

# 3. 检查端口是否监听
netstat -tlnp | grep 3000

# 4. 测试本地访问
curl http://localhost:3000/api/health

# 5. 检查防火墙
# 宝塔面板 → 安全 → 确认端口已开放

# 6. 检查阿里云安全组
# 阿里云控制台 → ECS → 安全组 → 添加入站规则
```

### 4. 反向代理配置后 502 错误

**解决方案**：

1. 确认容器正在运行：`docker compose ps`
2. 确认端口正确：反向代理目标应为 `http://127.0.0.1:3000`
3. 检查 Nginx 错误日志：`tail -f /www/wwwlogs/你的域名-error.log`
4. 重载 Nginx：`nginx -t && nginx -s reload`

### 5. 数据库连接失败

**问题**：使用 Supabase 时连接失败

**解决方案**：
```bash
# 检查环境变量
cat /www/wwwroot/warehouse-system/.env

# 确认 Supabase URL 和 Key 正确
# 重启容器
docker compose down
docker compose up -d

# 查看日志
docker compose logs warehouse-system | grep -i supabase
```

### 6. 内存不足

**问题**：服务器内存不足导致构建失败

**解决方案**：

```bash
# 创建 Swap 分区
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 验证
free -h
```

### 7. 如何修改默认登录密码

登录系统后：
1. 进入 "系统设置"
2. 点击 "用户管理"
3. 找到 admin 用户
4. 点击编辑，修改密码
